# Steam Trading Card Helper

Steam 卡牌助手是一款 Tampermonkey 用户脚本，用于扫描未完成的 Steam 徽章、批量查询并记录卡牌价格走势、估算升级成本、辅助购买，并自动合成已经收集齐全的徽章。

> **v2.3.0**：新增订单墙识别与保守/平衡/抢单自动定价，自动模式使用独立设置 profile，并加入最低卖价保护和确定性正负样本测试。

## 功能

- 扫描 Steam 徽章页，找出仍需补卡的游戏
- 批量查询卡牌最低售价、中位价格与成交量
- 从扫描结果、订购缓存或手动 `market_hash_name` 保存关注物品，在同一总览查看卡图、走势、在售量、24 小时销量、现价和较上次波动
- 估算补全一套、完整一套和升至目标等级的成本
- 卡牌价格扫描可切换闪卡模式，按闪亮徽章页面与 Lv1 满级逻辑计算
- 订购卡牌页实时接收扫描结果，可缓存候选卡组、手动输入 AppID、重新计算并提交长期订购单
- 订购缓存可设置保留天数，列表显示缓存天数，也可选择扫描时跳过缓存内结果
- 价格预测提前跳过明显超出上限的卡组，减少请求和等待
- 缺少最低售价时使用 `median_price`，价格信息大量缺失时使用保守公式估算
- 四种购买数量逻辑：补全单套、补至五级、购买单套、购买五套；闪卡结果会按 Lv1/一套自动封顶
- 打开 Steam multibuy 后自动填写数量和价格
- 多选扫描结果，重新计算持有量后批量提交长期订购单
- 订购价格可选在售最低、平均价格或求购最高，并支持正负买价调整
- 自动定价可识别最高买价附近的订单墙，按保守、平衡或抢单策略计算价格，并使用独立的策略与调整值 profile
- 提交订购单前自动抵扣已有订单数量
- 扫描可立即合成的徽章，批量选择合成次数
- 合成模式可选逐级升级或一次提交当前可合成最大次数
- 在多余物品处理页选择卡牌、背景或表情，按库存格子展示扫描结果并支持整格选中
- 左侧信息侧栏，快速查看等级经验、宝石数量、宝石袋价格、相对上次刷新的价格波动和分解临界点
- 检测可分解的背景/表情，按 Steam 税后收入比较卖出和分解成宝石的价值
- 扫描、合成和多余物品处理日志区域可拖动调整高度
- 手动、自动和固定游戏/AppID 黑名单管理
- 自动识别 Steam 钱包/市场页面币种；识别失败时使用设置中的回退币种，并在界面明确显示来源
- 市场记录与订购缓存按币种隔离，旧版 CNY 订购缓存会自动迁移

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/)。
2. 打开脚本安装链接：

   <https://raw.githubusercontent.com/SpaceSyt/Steam-Trading-Card-Helper/master/steam-trading-card-helper.user.js>

3. 打开任意 Steam 徽章页，例如 `steamcommunity.com/id/xxx/badges/`。
4. 点击页面中的 **Steam Trading Card Helper** 按钮。

新用户首次打开会看到一页精简说明。关闭后不会再次自动显示，可在 **设置** 页点击 **重新查看使用说明**。

## 基本使用

1. 打开 **卡牌价格扫描**，设置单套卡牌价格上限、购买逻辑、购买价格和买价调整；需要查闪卡时勾选 **闪卡模式**。
2. 点击 **开始扫描**。
3. 扫描完成后：
   - 点击 **购买**，进入 multibuy 并自动填写数量和价格。
   - 勾选多个结果，按需 **重新计算**，再 **提交订购单**。

## 订购卡牌

1. 打开 **订购卡牌** 页签查看价格扫描实时写入的候选卡组。
2. 也可以手动输入 AppID，例如 `4761370`，点击 **读取并加入** 直接查价并加入缓存。
3. 购买价格和买价调整与扫描页实时同步；勾选缓存结果后，可在此页 **重新计算** 或 **提交订购单**。

缓存天数默认 3 天；列表中的“天数”列与黑名单一致，`0` 表示今天，`1` 表示已缓存一天。设置页可开启 **扫描时跳过缓存内结果**，避免重复查价。

## 价格走势

1. 打开 **价格走势** 页签，从扫描结果或订购缓存选择卡牌并点击 **添加**；也可以切换到手动输入，填写例如 `753-Sack of Gems` 的 `market_hash_name` 后添加。
2. 每个已保存物品占一行：左侧图片和名称会打开对应 Steam Market listing 页面，中间显示迷你折线，右侧显示在售数量、24 小时销量、现价和较上次价格采样的波动。
3. 选择 **24 小时**、**7 天**、**30 天** 或 **全部** 可统一切换所有迷你折线的范围；“全部”指本地保留期内的全部记录。
4. 点击 **刷新全部价格** 可按需刷新所有已保存物品。正常的价格扫描、重新计算和最高买价读取也会继续写入历史，但不会自动把物品加入总览。

“24 小时销量”使用最新 `priceoverview.volume` 快照，不会累加多个滚动窗口；“现价”和“较上次”只比较连续两次同源最低卖价，避免 listing 与 priceoverview 的同轮响应制造假波动。刚添加或首次查看时通常只有零到一个样本，需要继续查价后才会形成折线和变化。行最角落的 **删除** 只在悬停或键盘聚焦时显示；它只把物品移出总览，保留已有价格历史，重新添加即可恢复折线。v2.2.0 不会在后台轮询；Steam 远端价格历史回填将在真实登录响应和限流行为验证后再启用。

## 徽章合成

1. 打开 **徽章合成** 页签，选择合成模式和最大徽章页数。
2. 扫描后选择要合成的徽章；逐级升级模式可调整次数，也可批量设为一次、最大或清零。
3. 检查确认清单后开始合成。

- **逐级升级**：每一级单独提交，便于准确记录进度，默认使用此模式。
- **一次升满**：按当前可合成最大次数一次提交；普通徽章会受等级上限限制，活动/促销徽章按库存套数放开。
- 合成请求默认间隔 500ms。
- 如果返回 429 或请求结果不确定，脚本会立即停止，不会自动重复合成。

## 多余物品处理

1. 从 Steam 社区徽章页或库存页打开脚本，进入 **多余物品处理** 页签。
2. 在第一行选择 **卡牌**、**背景** 或 **表情**。
3. 选择 **卡牌** 时点击 **开始检测**，脚本会读取社区库存中的集换式卡牌，再按库存里实际拥有的游戏读取对应徽章页。
4. 选择 **背景** 或 **表情** 时点击 **扫描可分解物品**，脚本会读取社区库存、识别可转宝石物品，并查询可出售物品的市场价。

结果会以类似 Steam 原生库存的格子展示，只显示扫描结果；点击任意格子位置即可选中或取消选中，也可对当前可见结果全选，并可筛选只显示可交易项目。卡牌结果会保留卡牌名、库存数量、升满预留数量、多余数量、可出售/可交易数量、宝石值、市场成交量和可用于处理的资产 ID。成交量为 0 的卡牌会以黄色覆盖；启用 **宝石比较** 后，脚本会按宝石袋和卡牌出售的税后到手价比较，以绿色覆盖分解成宝石更值的卡牌。普通徽章按 Lv5 计算，闪亮徽章按 Lv1 计算；活动/促销徽章未到 Lv1 时不列入多余处理，达到 Lv1 后不再预留后续等级所需卡牌。

背景/表情会先按同一种物品聚合并扣除设置中的默认保留数量；点数商店类不可交易副本默认不参与重复计算，可在设置页开启。建议会按宝石袋市场价反推卖家税后到手，再换算每件物品分解成宝石后的价值；物品市场价也按卖出税后到手比较。选中项目后可使用 **出售** 直接提交 Steam 市场上架请求，出售价格可选在售最低、平均价格或求购最高，并支持正负售价调整；确认窗口会显示买家价格和税后到手价。也可使用 **转化宝石**，脚本会先读取 Steam 当前宝石值，再带期望宝石值提交转化请求。出售或转化宝石前请仔细检查确认窗口，物品销毁不可逆，市场上架可能仍需要 Steam 手机应用确认。

## 默认配置

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 识别失败时的回退币种 | CNY | 优先使用 Steam 钱包或市场页面币种；只有无法识别时才使用此设置 |
| 单套卡牌价格上限 | ¥5 | 完整一套卡牌最低售价之和的上限 |
| 购买卡牌逻辑 | 补至五级 | 普通卡按当前徽章等级和持有量补至五级；活动/促销徽章按当前等级继续向后计算；闪卡结果按 Lv1 封顶 |
| 购买价格 | 在售最低 | 长期订购单的价格来源 |
| 买价调整 | ¥0 | multibuy 和长期订购单的价格加减值，允许负数 |
| 最大徽章页数 | 1 | 扫描的最大页数 |
| 包含掉落 | 关闭 | 是否包含仍有卡牌掉落的游戏 |
| 闪卡模式 | 关闭 | 在卡牌价格扫描内改用闪亮徽章 `?border=1` 页面，按 Lv1 作为满级目标 |
| 订购卡牌缓存 | 3 天 | 订购页缓存扫描/手动加入的查价结果，超期自动删除 |
| 扫描时跳过缓存 | 关闭 | 价格扫描遇到订购缓存内仍有效的结果时跳过重复查价 |
| 价格预测提早跳过 | 开启 | 对明显超限的卡组提前停止查询 |
| 预测跳过时加入自动黑名单 | 关闭 | 同时开启自动游戏黑名单后，预测价格超过其阈值时加入普通黑名单 |
| `priceoverview` 间隔 | 330ms | 相邻请求的启动时间间隔 |
| 主动冷却 | 每 20 次暂停 53s | 降低触发 Steam 429 的概率 |
| 显示无结果日志 | 关闭 | 是否显示常规无结果信息 |
| 合成模式 | 逐级升级 | 逐级提交，或一次提交当前可合成最大次数 |
| 合成请求间隔 | 500ms | 请求之间的等待时间 |
| 多余物品处理类型 | 卡牌 | 可切换卡牌、背景或表情 |
| 只显示可交易 | 关闭 | 仅显示扫描结果中至少有 1 件可交易资产的项目 |
| 宝石比较 | 关闭 | 按税后价值标记分解成宝石更值的多余卡牌 |
| 多余物品出售价格 | 在售最低 | 多余物品处理页提交出售时的价格来源 |
| 多余物品售价调整 | ¥0 | 上架出售价格加减值，允许负数 |
| 只显示建议分解 | 开启 | 背景/表情扫描默认只显示税后比较后更适合分解的物品 |
| 背景/表情默认保留 | 1 份 | 每种物品扣除保留数量后才视为多余 |
| 重复计算包含点数商店物品 | 关闭 | 是否让不可交易且不可上架的点数商店类副本参与重复数量计算 |

`priceoverview` 使用固定启动间隔：脚本会从 330ms 中扣除本次响应耗时。例如响应耗时 120ms，则再等待约 210ms；响应超过 330ms 时不额外等待。请求仍然串行执行，不会并发轰炸 Steam。

## 货币与市场缓存

- 当前版本已验证 CNY（Steam currency `23`）、USD（`1`）和 HKD（`29`）的两位小数规则；港币按 `HK$ 1.23` 显示，请求使用 `currency=29`。手续费继续使用 Steam/发行商百分比规则。其他可识别币种会保留实际 currency ID，等待真实样本完善。
- 识别顺序为 Steam 钱包上下文、页面上下文、用户设置回退。界面底部会显示当前实际币种及识别来源。
- `priceoverview`、挂牌页订单簿、订单直方图和价格历史使用同一市场记录结构；缺失价格保存为 `null`，不会与零价格混淆。
- 市场缓存按应用、物品、币种和来源隔离，默认保留 30 天且最多 5000 条。订购缓存按币种分区；从 v2.0.5 升级时，旧记录会迁入 CNY 分区，黑名单配置保持不变。
- 价格历史使用独立 schema，按物品、币种、来源和一小时采样桶隔离；默认保留 365 天、全局最多 20,000 条，展示时最多聚合为 480 个点。
- 切换回退币种后应重新查价；旧币种缓存仍会保留，但不会参与当前币种的计算或订单提交。

设置页提供 **重新查看使用说明**、**清除缓存** 和 **恢复默认设定**。清除缓存只清理订购卡牌缓存；恢复默认设定会保留游戏/AppID 黑名单和订购缓存。

如果连续出现 429，Steam 可能已临时限制当前 IP。建议等待至少半小时或者更换 IP 后再继续；卡牌价格扫描仍会触发强制冷却并重试。

开发检查可运行 `npm test`（固定夹具与基础设施测试）和 `npm run build`（生成用户脚本）。

v2.2.0 生产 userscript 为 281,449 B、481 行，相对 v2.1.5 基线增加 37,583 B（15.41%）；压缩 CSS 为 25,453 B，增加 4,681 B。增量来自独立历史与已保存物品存储的损坏保护、采样/保留边界、市场元数据解析、共享批量写入及逐行 Canvas 图表，已复核且仍低于 v2.x 的 300 KiB 软上限。

v2.3.0 生产 userscript 为 290,909 B、489 行；新增严格订单深度解析、近端墙检测、三档策略、独立自动定价 profile 与确认依据展示，仍低于 v2.x 的 300 KiB 软上限。

## 购买价格

- **在售最低**：使用扫描得到的 `lowest_price`，通常更容易立即成交。
- **平均价格**：使用 Steam 返回的 `median_price`。
- **求购最高**：提交前读取当前市场最高买单，通常需要等待卖家成交。

勾选 **自动定价模式** 后，购买价格和买价调整会切换到独立 profile：

- **保守**：有墙取近端订单墙底部；无墙取最高买价减 0.01。
- **平衡**：有墙取近端订单墙顶部；无墙取最高买价。
- **抢单**：取最高买价加 0.01。

自动 profile 的买价调整在策略价格之后生效，最终价格不会达到或超过最低卖价；订单簿解析失败时跳过该物品，不会按未知价格提交。

手动 multibuy 始终以在售最低为基础；购买价格选项只影响自动提交的长期订购单。

扫描页与订购页实时同步当前 profile。手动和自动 profile 的购买价格、买价调整互不覆盖；关闭自动定价后会恢复先前的手动设置。

如果调整后的价格低于 Steam 当前货币允许的最低购买价，脚本会在确认清单中提示并自动使用最低允许价格。所选价格缺失时会跳过该卡牌，不会静默改用其他价格。

## 价格估算

- 有 `lowest_price` 时使用最低售价。
- 没有最低售价但有 `median_price` 时，用中位价格参与满级估算。
- 如果至少 50% 的卡牌没有任何价格，而同套仍有可用价格，则用已知单价的几何平均数估算缺失卡牌。
- 使用中位价格、公式估算或存在未计价卡牌时，满级估算显示为黄色。
- 价格预测只在样本成交量和价差满足条件时启用，并要求预测结果超过价格上限的 105% 才提前跳过。
- 开启“预测跳过时加入自动黑名单”后，预测结果还必须超过自动游戏黑名单的独立价格阈值才会加入，不会仅因超过扫描上限而误加。

## 游戏/AppID 黑名单

- 手动按 AppID 添加或删除游戏。
- 自动游戏黑名单可将扫描时超过指定价格的游戏 AppID 加入黑名单。
- 自动游戏黑名单的开关和价格上限同时显示在黑名单页与设置页，修改任意一处都会立即同步。
- 固定游戏黑名单不会被自动清理。
- 普通游戏黑名单支持清理超过保存期限的条目。
