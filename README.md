# Steam Badge Helper

Tampermonkey 用户脚本，在 Steam 徽章页面扫描有未完成进度的游戏徽章，批量查询卡牌价格，计算卡牌套装成本，一键跳转 Steam 批量购买页面并自动填入数量和价格。

> **v1.4.2-rc.1**：优化扫描汇总显示，新增“求购最高”订购价格来源。

## 功能

- 自动扫描徽章页面，找出有未完成进度的游戏
- 批量查询卡牌市场价格（含最低价与中位数）
- 可选价格预测提早跳过（默认开启）
- 满级价格估算（绿色、灰色、红色表示成交量，黄色表示价格信息不完整）
- 一键跳转 multibuy 批量购买
  - 自动按商品链接精确匹配并填入数量和价格（最低价 + 买价调整），不受 Steam 商品行排序影响
  - 四种购买逻辑可选：补全单套 / 补至五级 / 购买单套 / 购买五套
  - multibuy 页恢复默认价格按钮
- 结果行支持多选操作
  - 手动购买后重新读取持有量和价格
  - 按在售最低、平均价格或求购最高批量提交长期订购单
  - 自动抵扣已有订购单数量，避免重复占用钱包余额
- 黑名单管理
  - 查询游戏名称 + 添加/移除 AppID
  - 自动黑名单：扫描时超限自动加入（可选开关）
- 点击结果行空白区域 → 新标签页打开徽章页面

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 浏览器扩展
2. 打开脚本的 GitHub raw 链接，Tampermonkey 会自动弹出安装提示：
   ```
   https://github.com/SpaceSyt/Steam-Badge-Helper/raw/master/steam-badge-helper.user.js
   ```
3. 安装后打开任意 Steam 徽章页 (`steamcommunity.com/id/xxx/badges/`)，页面右侧会出现 **Steam Badge Helper** 按钮

## 使用说明

1. 点击入口按钮打开助手面板
2. 设置 **单套卡牌价格上限**、**购买卡牌逻辑**、**购买价格**、**买价调整**
3. 点击 **开始扫描**，等待扫描完成
4. 点击结果行中的 **购买** → multibuy 页面自动打开并填好数量和价格 → 确认下单
5. 也可以勾选多个结果，先 **重新计算** 持有量，再 **提交订购单** 挂长期订单

## 配置项

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 单套卡牌价格上限 | ¥5 | 全套卡牌最低价之和的上限 |
| 购买卡牌逻辑 | 补至五级 | multibuy qty 计算方式；按当前徽章等级补至五级 |
| 购买价格 | 在售最低 | 自动提交长期订购单使用的价格来源，可选在售最低、平均价格或求购最高；不影响手动 multibuy |
| 买价调整 | ¥0 | 手动 multibuy 和长期订购单的买价加减值，允许负数 |
| 最大徽章页数 | 1 | 扫描的最大页数 |
| 包含掉落 | 关 | 是否包含仍有掉落的游戏 |
| 价格预测提早跳过 | 开 | 扫描 2-4 张后保守预测全套价格，超过安全线时提前跳过 |
| 请求间隔 | 450ms | 每次 `priceoverview` 请求间隔 |
| 批暂停 | 20次/45s | 每 20 次 `priceoverview` 后主动冷却 |

`gamecard` 请求通常不需要调整，保持 `0ms` 即可。

## 长期订购单

扫描完成后，可以勾选结果行末尾的复选框：

1. **重新计算**会重新读取选中游戏的徽章等级、卡牌持有量、`lowest_price` 和 `median_price`。适合先用手动 multibuy 买入一部分，再计算剩余数量。
2. **提交订购单**根据当前购买逻辑计算目标数量，并使用“购买价格 + 买价调整”创建长期订购单。
3. 提交前会读取 Steam 市场中的现有订购单，并按 `marketHashName` 抵扣已有数量。
4. 脚本会展示逐卡确认清单；只有用户确认后才会顺序调用 Steam 的 `createbuyorder`。
5. 长期订购单不会像 multibuy 一样在约 5 秒后取消，会一直保留到成交或手动取消。

“在售最低”使用 `lowest_price`；“平均价格”使用 Steam 返回的 `median_price`；“求购最高”会在提交前按需读取当前最高买单价格。求购最高只查询仍需创建订购单的卡牌，不增加正常扫描请求。

如果调整后的价格低于 Steam 对当前货币设置的最低购买价，脚本会在确认清单中提示并自动使用最低允许价格。所选价格字段缺失时，该卡牌会被跳过，不会静默改用另一种价格。

## 提前价格预测

**价格预测提早跳过**默认开启。扫描每套卡牌时，脚本会在查到第 2、3、4 张后尝试预测全套价格：

1. 已查询卡牌使用实际最低价累加，不进行折扣。
2. 仅当所有样本成交量均大于 0，且最高价小于最低价的 2 倍时才预测。
3. 根据样本最低价与最高价计算剩余卡牌的保守均价：

   | 已查询数量 | 剩余均价估算 |
   |---|---|
   | 2 张 | `0.78 × (最低价 + 0.20 × 价差)` |
   | 3 张 | `0.80 × (最低价 + 0.30 × 价差)` |
   | 4 张 | `0.84 × (最低价 + 0.25 × 价差)` |

4. `预测全套价格 = 已查询卡牌实际总价 + 未查询数量 × 剩余均价估算`。
5. 只有预测价格超过设置上限的 **105%** 时才提前跳过，降低误判风险。

关闭该选项后，脚本只会根据已查询卡牌的实际累计价格判断是否超限。

## 价格缺失处理

Steam 对低访问量卡牌可能不会返回完整的 `priceoverview` 数据：

1. 有 `lowest_price` 时，使用最低出售价格。
2. 没有 `lowest_price`、但有 `median_price` 时，使用 `median_price` 参与满级价格估算；它不会覆盖 multibuy 的购买价格。
3. 如果至少 50% 的卡牌只返回 `success`、没有任何价格，而同套仍有可用价格：
   - 取已知卡牌有效单价的几何平均数；
   - 按每张缺失卡牌升至五级所需的数量补充满级估算；
   - 几何平均数比算术平均数更不容易被单张高价卡拉高。
4. 如果整套卡牌都没有任何可用价格，则无法建立估算基准，跳过该游戏。

使用 `median_price`、公式估算或存在未计价卡牌时，满级估算显示为黄色。鼠标悬停可查看具体原因。此类结果可能偏低，后续版本将通过市场详情页补充价格。

## 注意事项

- 仅支持人民币 (CNY)
- 满级价格为估算值，成交量低或 Steam 返回信息不完整时可能不准
- 不支持闪卡

## Steam 请求速率限制

Steam 市场 API 的具体速率限制官方并未公开。当前脚本默认每次 `priceoverview` 请求间隔 450ms，并在每 20 次请求后主动暂停 45 秒。如果仍收到 429，脚本会额外暂停 45 秒后重试，作为被动限流保护。

如果按 F12 打开开发者工具，所有 `priceoverview` 请求都返回 429，说明 IP 可能已被暂时限制。解决方法：等待几小时或更换 IP。扫描中连续 5 次 429 后日志区会显示提醒。
