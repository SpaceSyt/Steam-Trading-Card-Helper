# Steam Trading Card Helper

Steam 卡牌助手是一款 Tampermonkey 用户脚本，用于扫描未完成的 Steam 徽章、批量查询卡牌价格、估算升级成本、辅助购买，并自动合成已经收集齐全的徽章。

> **v1.5.6**：修复季节徽章读取方式，并在 Steam 点数商店余额下方增加入口。

## 功能

- 扫描 Steam 徽章页，找出仍需补卡的游戏
- 批量查询卡牌最低售价、中位价格与成交量
- 估算补全一套、完整一套和升至五级的成本
- 价格预测提前跳过明显超出上限的卡组，减少请求和等待
- 缺少最低售价时使用 `median_price`，价格信息大量缺失时使用保守公式估算
- 四种购买数量逻辑：补全单套、补至五级、购买单套、购买五套
- 打开 Steam multibuy 后自动填写数量和价格
- 多选扫描结果，重新计算持有量后批量提交长期订购单
- 订购价格可选在售最低、平均价格或求购最高，并支持正负买价调整
- 提交订购单前自动抵扣已有订单数量
- 扫描可立即合成的徽章，批量选择合成次数
- 合成模式可选逐级升级或调用 Steam 原生最大合成
- 购买当前季节徽章等级，支持逐级购买或一次买满至 40 级
- 手动、自动和固定黑名单管理

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/)。
2. 打开脚本安装链接：

   <https://raw.githubusercontent.com/SpaceSyt/Steam-Trading-Card-Helper/master/steam-trading-card-helper.user.js>

3. 打开任意 Steam 徽章页，例如 `steamcommunity.com/id/xxx/badges/`。
4. 点击页面中的 **Steam Trading Card Helper** 按钮。季节徽章也可以从 Steam 点数商店页面余额下方的入口打开。

新用户首次打开会看到一页精简说明。关闭后不会再次自动显示，可在设置页选择 **重新查看使用说明**。

## 基本使用

1. 设置单套卡牌价格上限、购买逻辑、购买价格和买价调整。
2. 点击 **开始扫描**。
3. 扫描完成后：
   - 点击 **购买**，进入 multibuy 并自动填写数量和价格。
   - 勾选多个结果，按需 **重新计算**，再 **提交订购单**。

## 徽章合成

1. 打开 **徽章合成** 页签，选择合成模式和最大徽章页数。
2. 扫描后选择要合成的徽章；逐级升级模式可调整次数，也可批量设为一次、最大或清零。
3. 检查确认清单后开始合成。

- **逐级升级**：每一级单独提交，便于准确记录进度，默认使用此模式。
- **一次升满**：按 Steam 页面给出的原生最大次数一次提交，适合普通多级徽章和季节徽章。
- 合成请求默认间隔 500ms；逐级模式按每一级等待，一次升满按每个徽章等待。
- 如果返回 429 或请求结果不确定，脚本会立即停止，不会自动重复合成。

## 季节徽章

1. 从徽章页或 Steam 点数商店页面打开脚本，进入 **季节徽章** 页签。
2. 确认当前等级、目标等级和购买模式。
3. 点击 **读取当前季节** 可从 Steam 点数商店读取当前季节徽章定义。
4. 点击 **开始购买**，检查确认窗口后提交。

- **逐级购买**：每次只购买 1 级，便于中途停止和记录进度。
- **一次买满**：一次提交目标等级与当前等级之间的全部等级。
- 目标等级限制为 1-40，默认从 0 级买到 40 级。
- 当前只自动读取官方点数商店暴露的当前季节徽章。旧年份没有稳定公开入口；如果已知旧季节 `defid`，可以填写自定义 `defid` 尝试，是否允许购买由 Steam 服务端决定。
- 购买会消耗 Steam 点数。请求超时、429 或结果不确定时脚本会停止，不会自动重复提交，请刷新点数商店确认实际等级。

## 默认配置

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 单套卡牌价格上限 | ¥5 | 完整一套卡牌最低售价之和的上限 |
| 购买卡牌逻辑 | 补至五级 | 按当前徽章等级和持有量补至五级 |
| 购买价格 | 在售最低 | 长期订购单的价格来源 |
| 买价调整 | ¥0 | multibuy 和长期订购单的价格加减值，允许负数 |
| 最大徽章页数 | 1 | 扫描的最大页数 |
| 包含掉落 | 关闭 | 是否包含仍有卡牌掉落的游戏 |
| 价格预测提早跳过 | 开启 | 对明显超限的卡组提前停止查询 |
| `priceoverview` 间隔 | 330ms | 相邻请求的启动时间间隔 |
| 主动冷却 | 每 20 次暂停 53s | 降低触发 Steam 429 的概率 |
| `gamecard` 间隔 | 0ms | 通常不需要额外延迟 |
| 合成模式 | 逐级升级 | 逐级提交，或一次提交 Steam 原生最大次数 |
| 合成请求间隔 | 500ms | 逐级模式按每级，一次升满按每个徽章等待 |
| 季节徽章购买模式 | 逐级购买 | 逐级购买，或一次购买至目标等级 |
| 季节徽章目标等级 | 40 | 点数商店季节徽章最高 40 级 |
| 季节徽章购买间隔 | 500ms | 逐级购买时每级之间的等待时间 |

`priceoverview` 使用固定启动间隔：脚本会从 330ms 中扣除本次响应耗时。例如响应耗时 120ms，则再等待约 210ms；响应超过 330ms 时不额外等待。请求仍然串行执行，不会并发轰炸 Steam。

如果连续出现 429，Steam 可能已临时限制当前 IP。建议等待至少半小时或者更换 IP 后再继续；价格扫描仍会触发强制冷却并重试。

## 购买价格

- **在售最低**：使用扫描得到的 `lowest_price`，通常更容易立即成交。
- **平均价格**：使用 Steam 返回的 `median_price`。
- **求购最高**：提交前读取当前市场最高买单，通常需要等待卖家成交。

手动 multibuy 始终以在售最低为基础；购买价格选项只影响自动提交的长期订购单。

如果调整后的价格低于 Steam 当前货币允许的最低购买价，脚本会在确认清单中提示并自动使用最低允许价格。所选价格缺失时会跳过该卡牌，不会静默改用其他价格。

## 价格估算

- 有 `lowest_price` 时使用最低售价。
- 没有最低售价但有 `median_price` 时，用中位价格参与满级估算。
- 如果至少 50% 的卡牌没有任何价格，而同套仍有可用价格，则用已知单价的几何平均数估算缺失卡牌。
- 使用中位价格、公式估算或存在未计价卡牌时，满级估算显示为黄色。
- 价格预测只在样本成交量和价差满足条件时启用，并要求预测结果超过价格上限的 105% 才提前跳过。

## 黑名单

- 手动按 AppID 添加或删除游戏。
- 自动黑名单可将扫描时超过指定价格的游戏加入黑名单。
- 固定黑名单不会被自动清理。
- 普通黑名单支持清理超过保存期限的条目。
